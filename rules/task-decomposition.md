# Task Decomposition

Перед тем как браться за задачу, оцени её сложность. Если задача затрагивает более одного модуля/слоя, требует нескольких шагов, или её объём неочевиден — **не начинай выполнение сразу**.

## Когда требуется декомпозиция

- Задача затрагивает несколько файлов, модулей или слоёв архитектуры
- Есть архитектурный выбор с неочевидными trade-offs
- Задача включает и исследование, и изменения кода
- Объём работы неочевиден до начала

## Процесс

1. Запусти Plan-агента (`subagent_type: Plan`) для анализа и декомпозиции.
2. Сохрани план в `./swarm-report/<slug>-plan.md` (формат ниже).
3. Покажи план пользователю — жди подтверждения перед стартом.
4. Выполняй шаги по одному, отмечая `[x]` после завершения каждого.
5. Параллельные шаги (без зависимостей друг от друга) — запускай одновременно в одном сообщении.

## Формат плана

Файл: `swarm-report/<slug>-plan.md`

```markdown
# Plan: <название задачи>

## Goal
<одна строка — что должно быть достигнуто в результате>

## Steps

### Step 1 — <название шага>

| Field | Value |
|---|---|
| Agent | Explore / Plan / kotlin-engineer / compose-developer / ... |
| Complexity | low / medium / high |
| Depends on | — / Step N |
| Parallel with | — / Step N |

**Description:** что конкретно нужно сделать в этом шаге.

**Context / inputs:** какие файлы, модули, данные или результаты предыдущих шагов нужны агенту.

**Instructions:** конкретные действия — что читать, что менять, каким паттернам следовать, на что обратить внимание.

**Acceptance criteria:**
- [ ] критерий 1 — конкретная проверяемая характеристика результата
- [ ] критерий 2

**Output / handoff:** что передаётся следующему шагу или является финальным результатом шага.

---

### Step 2 — <название шага>
...
```

## Исключения (декомпозиция не нужна)

- Однофайловые правки с очевидным scope
- Ответы на вопросы без изменений кода
- Задачи, укладывающиеся в одно действие одного агента
