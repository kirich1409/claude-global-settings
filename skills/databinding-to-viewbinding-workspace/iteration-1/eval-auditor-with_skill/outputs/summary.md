# Migration Summary: RefRequestAuditorFragment (DataBinding to ViewBinding + StateFlow)

## Files Changed

### New File
- `reference/reference-impl/src/main/kotlin/by/st/alfa/ib2/reference_impl/internal/presentation/edit/requestauditor/ErrorState.kt`
  - Created `ErrorState` data class with `isActive` and `errorText` fields, plus `enabled()`/`disabled()` companion factory methods.
  - Replaces the role of `ObservableError.ErrorState` for the migrated StateFlow fields.

### Modified Files

1. **`RefRequestAuditorDataModel.kt`**
   - Removed `ObservableField` and `ObservableError` imports.
   - Converted `address` from `ObservableField<String>()` to `MutableStateFlow("")`.
   - Converted `addressError` from `ObservableError()` to `MutableStateFlow(ErrorState())`.
   - Converted `attachmentError` from `ObservableError()` to `MutableStateFlow(ErrorState())`.
   - Converted `copiesNumber` from `ObservableField<String>()` to `MutableStateFlow("")`.
   - Converted `copiesNumberError` from `ObservableError()` to `MutableStateFlow(ErrorState())`.
   - Base class fields (`notifyEmail`, `email`, `emailError`, `confirmText`) left as `ObservableField`/`ObservableBoolean`/`ObservableError` since `BaseRefDataModel` is shared by 5+ other screens.

2. **`fragment_reference_auditor_request.xml`**
   - Removed `<layout>` wrapper and `<data>` section with all variable declarations.
   - Removed all `@{...}` and `@={...}` binding expressions (`bind_alfaInputText`, `bind_error`, `bind_hasError`, `bind_inputChangeListener`, `bind_singleClickListener`, `bind_inputVisibility`, `onFocusChangeListener`, `onAddClickListener`, `android:text`).
   - Added `android:id` to views that lacked one but need programmatic access: `frar_address`, `frar_copies_number`, `frar_email_switch`.
   - Kept all static XML attributes (`alfaInputHint`, `alfaInputMaxLength`, etc.).

3. **`RefRequestAuditorFragment.kt`**
   - Replaced DataBinding inflation with ViewBinding delegate (`viewBinding(FragmentReferenceAuditorRequestBinding::bind)`).
   - `onCreateView` now inflates via `FragmentReferenceAuditorRequestBinding.inflate(...)`.
   - Moved all setup logic to `onViewCreated` organized into four methods:
     - `setupListeners()`: wires text change listeners (using `AlfaInputView.textChanges` and `AlfaSwitchInputView.addTextChangedListener`), focus listeners, click listeners (`setSingleClickListener`, `setOnAddClickListener`, `setOnCheckedChangeListener`), and keyboard visibility observer.
     - `setupObservers()`: collects `MutableStateFlow` fields (`address`, `addressError`, `copiesNumber`, `copiesNumberError`) using `repeatOnLifecycle(STARTED)` for lifecycle-aware collection. Includes guards against two-way binding loops.
     - `setupBaseModelObservers()`: observes base class `ObservableField`/`ObservableBoolean`/`ObservableError` fields (`confirmText`, `notifyEmail`, `email`, `emailError`) via `addOnPropertyChangedCallback`, with initial value sync.
     - `observeLiveDataActions()`: all LiveData observations from ViewModel (unchanged from original).
   - Replaced `ViewDataBinding.getFirstViewWithError()` with a local `getFirstViewWithError(LinearLayout)` function that iterates children and checks `hasError`.
   - Added private extension functions: `AlfaInputView.applyError(ErrorState)` and `AlfaSwitchInputView.applyObservableError(ObservableError.ErrorState)`.
   - Removed `FragmentReferenceAuditorRequestBinding` DataBinding usage; now uses ViewBinding-generated class (same class name, but generated by ViewBinding instead of DataBinding).

4. **`RefRequestAuditorViewModel.kt`**
   - Replaced `dataModel.attachmentError.observe { ... }` (ObservableError extension) with a coroutine flow: `dataModel.attachmentError.filter { it.isActive }.onEach { ... }.launchIn(viewModelScope)`.
   - Updated `takeFormData()`: `dataModel.address.get()` -> `dataModel.address.value.ifEmpty { null }`, `dataModel.copiesNumber.get()?.toIntOrNull()` -> `dataModel.copiesNumber.value.toIntOrNull()`.
   - Updated `processSuccessInitial()`: `address.set(...)` -> `address.value = ...`, `copiesNumber.set(...)` -> `copiesNumber.value = ...`.
   - Updated `onAddressFocusChanged()`: `dataModel.address.get()` -> `dataModel.address.value.ifEmpty { null }`.
   - Updated `onCopiesNumberFocusChanged()`: `dataModel.copiesNumber.get()?.toIntOrNull()` -> `dataModel.copiesNumber.value.toIntOrNull()`.
   - Updated `createAttachment()`: `dataModel.attachmentError.enable(errorText)` -> `dataModel.attachmentError.value = ErrorState.enabled(errorText)`.
   - Removed import of `by.st.alfa.ib2.document.presentation.observe`.

5. **`RefRequestAuditorValidationHelper.kt`**
   - Updated `handleAddressValidationResult()`: `dataModel.addressError.enable(...)` -> `dataModel.addressError.value = ErrorState.enabled(...)`, `dataModel.addressError.disable()` -> `dataModel.addressError.value = ErrorState.disabled()`.
   - Updated `handleFileValidationResult()`: same pattern for `attachmentError`.
   - Updated `handleCopyCountValidationResult()`: same pattern for `copiesNumberError`.
   - Removed `ObservableError` import.

## Design Decisions

1. **BaseRefDataModel NOT migrated**: `BaseRefDataModel` is shared by 6+ DataModel subclasses across multiple reference screens. Converting it would break all other screens. Instead, base class fields are observed manually via `addOnPropertyChangedCallback` in the Fragment.

2. **ErrorState placement**: Created in the `requestauditor` package since it's only used by this screen's migrated fields. Can be promoted to a shared location when more screens are migrated.

3. **Two-way binding loop prevention**: Flow collectors guard against loops with `if (view.text != flowValue)` checks before setting text.

4. **Build config unchanged**: `dataBinding = true` remains in `build.gradle.kts` since other screens in `reference-impl` still use DataBinding. `viewBinding = true` was already enabled.
