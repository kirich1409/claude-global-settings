---
name: UI design review required before closing screens
description: Always do a design review (screenshot + fixes) before marking any UI screen as done
type: feedback
---

After any UI changes, always take a screenshot of the running app, review the design, and fix issues before considering the screen complete.

**Why:** User expects visual quality to be verified, not just "it compiles and works".

**How to apply:** After every UI change → install APK → screenshot via mobile MCP → review → fix → repeat until the screen looks good.
